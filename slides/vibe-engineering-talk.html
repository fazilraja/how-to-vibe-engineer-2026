Phase 1

Research

Map the system before you move

  • Input: Diagrams, docs, Slack threads
  • Action: Ask AI to map components
  • Checkpoint: Correct the analysis manually
  • Output: A validated research document
17 / 22
Phase 2

The Spec

Paint by numbers — plan BEFORE writing code

  • Function signatures, data flow, type definitions
  • Edge cases and error handling spelled out
  • Integration points mapped
  • Rule: "Go do this" should be the only instruction needed
18 / 22
Phase 3

Implementation

Simple execution because the plan is solid

  • No "wait, actually" loops
  • No abandoned code paths
  • No spaghetti
  • The agent executes. You verify.
